  (defun db-create-github-task (task-id-str text &key project-id doc-path context)
    "Create a task with explicit task_id, source='github', and doc_path.
     Used by GitHub sync to insert issues with dedup-safe task IDs."
    (handler-case

               (esc-text (db-escape text))
               (esc-doc-path (db-escape (or doc-path "github/unknown")))

               (sql (format nil
                      "INSERT INTO tasks (task_id, text, status, doc_path, line_number, raw_line, ~
                       source, project_id, context) ~
                       VALUES (~a, ~a, 'open', ~a, 0, ~a, 'github', ~a, ~a) ~
                       ON CONFLICT (task_id) DO NOTHING ~
                       RETURNING id, task_id"
                      esc-task-id esc-text esc-doc-path esc-text esc-proj esc-context)))
          (db-query-single sql))
      (error (e)
        (format t "  [db-create-github-task] error: ~a~%" e)

  <action>
  Create `/opt/project-noosphere-ghosts/lisp/util/github.lisp` with the following structure:

  ```lisp
  (in-package :af64.utils.github)
  ```

  **Environment/config:**
  - `*github-token*` — defparameter, reads `(uiop:getenv "GITHUB_TOKEN")` at load time
  - `*github-api-base*` — defparameter, `"https://api.github.com"`

  **Helper: `github-headers`** — returns list of header cons cells:
  - `("Authorization" . "Bearer {*github-token*}")`
  - `("Accept" . "application/vnd.github+json")`
  - `("User-Agent" . "AF64-Noosphere")`
  - `("X-GitHub-Api-Version" . "2022-11-28")`

  **Core API wrappers:**
  - `github-api-get (path)` — calls `(http-get (format nil "~a~a" *github-api-base* path) :headers (github-headers))`. Parses response body with parse-json. Returns (values parsed-json http-status). Logs errors with format t.
  - `github-api-post (path body-alist)` — calls http-post with (encode-json body-alist) and Content-Type header added. Same return pattern.
  - `github-api-patch (path body-alist)` — same pattern with http-patch.

  **Helper: `parse-github-url (url)`** — extracts owner and repo from URL.
  Handle both `https://github.com/owner/repo` and `https://github.com/owner/repo.git` and trailing slashes.
  Use string operations (search, subseq, position). Return `(values owner repo)`.
  Return `(values nil nil)` if URL doesn't match pattern.

  **Helper: `github-task-id (repo issue-number)`** — returns `(format nil "gh-~a-~a" repo issue-number)`.

  **`sync-github-issues ()`:**
  1. Call `(db-get-github-projects)` to get projects with github_url
  2. For each project:
     a. Extract owner/repo via parse-github-url from `(gethash :github-url project)` (note: JSON parser converts underscores to hyphens in keys)
     b. Skip if parse returns nil
     c. GET `/repos/{owner}/{repo}/issues?state=all&per_page=100` (fetch both open and closed)
     d. Check HTTP status is 200, skip on error
     e. Iterate over issues array (each is a hash-table after parse-json)
     f. SKIP pull requests: if `(gethash :pull-request issue)` is non-nil, skip (GitHub API returns PRs in issues endpoint)
     g. Build task-id via `(github-task-id repo (gethash :number issue))`
     h. Check if task exists: `(db-find-task-by-task-id task-id-str)`
     i. If NOT exists: call `(db-create-github-task task-id-str title :project-id proj-id :doc-path doc-path :context body)`
        - title = `(gethash :title issue)`
        - body = `(gethash :body issue)` (may be nil)
        - doc-path = `(format nil "github/~a/~a" owner repo)`
        - proj-id = `(gethash :id project)`
     j. If exists AND status changed: update via `(db-update-task existing-id :status new-status)`
        - GitHub state "open" maps to task status "open"
        - GitHub state "closed" maps to task status "done"
        - Only update if `(gethash :status existing)` differs from mapped status
     k. Log: `(format t "  [github-sync] ~a/~a: ~a new, ~a updated~%" owner repo new-count update-count)`

  **`push-tasks-to-github ()`:**
  1. Get all github-linked projects via db-get-github-projects
  2. For each project, query tasks: use direct SQL via db-query:
     ```sql
     SELECT id, task_id, text, status, source FROM tasks
     WHERE project_id = {proj-id} AND source != 'github' AND status = 'open'
     ```
  3. For each task, check if a corresponding GitHub issue already exists by checking if task_id starts with "gh-". If it does, skip.
  4. Otherwise, create issue via github-api-post to `/repos/{owner}/{repo}/issues` with body `(json-object :title (gethash :text task))`
  5. On success (status 201), extract issue number from response, update task in DB:
     - Set task_id to `(github-task-id repo issue-number)` and source to 'github' via direct SQL:
       ```sql
       UPDATE tasks SET task_id = '{new-task-id}', source = 'github', updated_at = NOW() WHERE id = {task-integer-id}
       ```
  6. Log each push.

  **`sync-task-status ()`:**
  1. Get all github-linked projects
  2. For each project, query github-sourced tasks: direct SQL via db-query:
     ```sql
     SELECT id, task_id, text, status FROM tasks
     WHERE project_id = {proj-id} AND source = 'github'
     ```
  3. For each task, extract repo and issue number from task_id (parse "gh-{repo}-{number}" — split on "-", repo is middle, number is last)
  4. GET `/repos/{owner}/{repo}/issues/{number}` from GitHub
  5. Compare states:
     - If GitHub is "closed" but task is "open" → `(db-update-task task-int-id :status "done")`
     - If GitHub is "open" but task is "done" → PATCH GitHub issue to close it: `(github-api-patch path (json-object :state "closed"))`
  6. Log changes.

  **`sync-all ()`:**
  1. Check `*github-token*` is non-nil and non-empty. If not, log warning and return.
  2. `(format t "[github-sync] Starting sync cycle...~%")`
  3. Call sync-github-issues wrapped in handler-case
  4. Call push-tasks-to-github wrapped in handler-case
  5. Call sync-task-status wrapped in handler-case
  6. `(format t "[github-sync] Sync cycle complete.~%")`

  Each sub-call should be independently wrapped so one failure doesn't abort the others.

  **Error handling pattern:** Every function uses handler-case wrapping the body, catching `error` condition, logging with `(format t "  [github-{fn-name}] error: ~a~%" e)`, and returning nil or empty vector on failure. Same pattern as all db-* modules.

  **IMPORTANT NOTES:**
  - The JSON parser converts underscores to hyphens in keys, so GitHub's `pull_request` becomes `:pull-request`, `github_url` becomes `:github-url`, etc.
  - Use `(uiop:getenv "GITHUB_TOKEN")` not reading from config file — same pattern as other env vars in the codebase.
  - The tasks table has a UNIQUE constraint on task_id, and db-create-github-task uses ON CONFLICT DO NOTHING for safe dedup.
  - GitHub issue numbers are integers in the API response, so `(gethash :number issue)` returns an integer — format with ~a is fine.
