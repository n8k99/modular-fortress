    **yaml.lisp** — Create `/opt/project-noosphere-ghosts/lisp/util/yaml.lisp` with `(in-package :af64.utils.yaml)`. Implement:
    - `split-lines`: Split text on newlines, return list of strings
    - `unquote-yaml-string`: Strip leading/trailing double quotes from a string
    - `parse-simple-yaml`: Parse minimal YAML (comments, `key: value` scalars, `- "value"` string lists) into a hash-table with string keys. The YAML structure is: `id: agent_name` scalar + `responsibilities:` list of quoted strings. All responsibility strings MUST be double-quoted in YAML because InnateScipt expressions contain `!`, `{`, `}`, `[`, `]` which are YAML special characters (per D-02, Pitfall 2 from RESEARCH.md).
    - `serialize-simple-yaml`: Convert hash-table back to YAML string. Emit `id: value` then `responsibilities:` with `  - "expr"` per item.

    **ghost-capabilities.lisp** — Create `/opt/project-noosphere-ghosts/lisp/runtime/ghost-capabilities.lisp` with `(in-package :af64.runtime.ghost-capabilities)`. Implement:
    - `*ghost-yaml-dir*`: defvar pointing to `"/opt/project-noosphere-ghosts/config/agents/"`
    - `ghost-yaml-path (agent-id)`: Return pathname `*ghost-yaml-dir*/{agent-id}.yaml`
    - `load-ghost-capabilities (agent-id)`: Read YAML file via `uiop:read-file-string`, parse with `parse-simple-yaml`, return the `"responsibilities"` list. If file doesn't exist, return nil (per D-11 — caller falls back to tool-registry.json). Wrap in handler-case for file errors.
    - `format-capabilities-for-prompt (capabilities)`: When capabilities is non-nil, format as:
      ```
      YOUR CAPABILITIES (InnateScipt expressions you can use):
        - ![query_db]
        - ![pipeline_status]
      These define what you can do. When performing work, use these capabilities.
      ```
      Return nil when capabilities is nil/empty (so format ~@[] skips it in callers).
    - `validate-capability-list (capabilities)`: Validate each string via `validate-innate-expression`. Return list of (expression . valid-p) pairs. Log warnings for invalid expressions but do not reject the whole file — skip invalid ones with a warning.

    **packages.lisp** — Add two new package definitions:
    1. `af64.utils.yaml` package: `(:use :cl)`, export `parse-simple-yaml`, `serialize-simple-yaml`, `unquote-yaml-string`.
    2. `af64.runtime.ghost-capabilities` package: `(:use :cl)`, import `parse-simple-yaml` from `af64.utils.yaml`, import `uiop:read-file-string`, import `validate-innate-expression` from `af64.runtime.innate-builder` (use handler-case eval pattern like innate-builder package itself since innate may not be loaded). Export: `load-ghost-capabilities`, `format-capabilities-for-prompt`, `ghost-yaml-path`, `validate-capability-list`, `write-ghost-yaml`, `process-responsibility-mutations`.

    Place `af64.utils.yaml` after `af64.utils.http` (line 18). Place `af64.runtime.ghost-capabilities` package definition after the `af64.runtime.innate-builder` handler-case block (after line 246) and before `af64.runtime.action-planner` (line 248).
