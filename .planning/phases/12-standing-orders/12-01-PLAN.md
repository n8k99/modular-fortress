    **Step 3: Create cron-matcher.lisp** (new file, per D-05)
    Implement a standard 5-field cron matcher supporting: `*`, exact number, ranges (`N-M`), lists (`N,M,O`), and steps (`*/N`).

    The file must:
    - Be in package `:af64.runtime.cron-matcher`
    - Provide `split-cron-string` helper that splits on space characters
    - Provide `parse-cron-field` that returns a function (lambda) taking an integer, returning T/NIL
    - Provide `cron-matches-p` that takes a cron expression string plus minute, hour, dom, month, dow integers and returns T/NIL
    - Handle edge cases: extra whitespace in expression, day-of-week 0 and 7 both meaning Sunday
    - NOT handle timezone conversion (all expressions stored in UTC per research anti-pattern guidance)

    Implementation details for `parse-cron-field`:
    - `"*"` -> always returns T
    - `"*/N"` -> returns T when `(zerop (mod val N))`
    - `"N-M"` -> returns T when `(<= N val M)`
    - `"N,M,O"` -> returns T when val is a member of the list
    - `"N"` -> returns T when `(= val N)`
    - Use `parse-integer` for number parsing
    - Use a simple string split on #\- and #\, characters

    For `split-cron-string`, split on spaces and filter out empty strings (handles multiple spaces).

    For `cron-matches-p`:
    ```lisp
    (defun cron-matches-p (expr minute hour dom month dow)
      "Return T if 5-field cron expression matches given time components."
      ...)
    ```
    Split expr into 5 fields, parse each with `parse-cron-field`, funcall each matcher against the corresponding time component. Return T only if all 5 match.
