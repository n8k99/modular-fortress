**Change 1: Replace prev-stage-map in action-planner.lisp (lines 386-430)**

The function `load-predecessor-stage-output` currently has a 28-line local `prev-stage-map` alist in a `let*` on lines 389-416. Replace the entire function with:

```lisp
(defun load-predecessor-stage-output (goal-id current-stage)
  "Load the stage_notes from the predecessor pipeline stage via DB query.
   Returns the parsed JSONB hash-table (or nil if no predecessor found).
   Per D-15: prev-stage-map replaced by get-prev-stage DB lookup."
  (let* ((pipeline-name (detect-pipeline-type current-stage))
         (prev-stage (when pipeline-name (get-prev-stage pipeline-name current-stage))))
    (when (and prev-stage goal-id)
      (handler-case
          (let ((tasks (db-get-tasks-by-filter :goal-id goal-id :limit 50)))
            (when tasks
              (let ((task-list (if (vectorp tasks)
                                   (coerce tasks 'list)
                                   (when (listp tasks) tasks))))
                (dolist (t-item task-list)
                  (when (and (hash-table-p t-item)
                             (string-equal (gethash :STAGE t-item) prev-stage)
                             (gethash :STAGE-NOTES t-item))
                    (return (gethash :STAGE-NOTES t-item)))))))
        (error () nil)))))
```

To avoid circular dependency (action-planner should not import from action-executor), use `get-pipeline-type-for-stage` directly:

```lisp
(let* ((pipeline-name (get-pipeline-type-for-stage current-stage))
       (prev-stage (when pipeline-name (get-prev-stage pipeline-name current-stage))))
```

This requires adding `get-pipeline-type-for-stage` to the action-planner package imports in packages.lisp. Check if Plan 01 already added it; if not, add:
