The module must contain:

1. Package declaration: `(in-package :af64.runtime.tool-definitions)`

2. Cache variable:
```lisp
(defvar *tool-definition-cache* (make-hash-table :test #'equal)
  "Cached tool definitions from area_content. Refreshed per tick.")
```

3. `load-tool-definitions` function:
   - SQL: `SELECT title, body, metadata FROM area_content WHERE content_type = 'tool' AND status = 'active'`
   - Uses `db-query` (from db-auxiliary package, same as pipeline-definitions.lisp)
   - For each row: parse metadata JSONB if string, set :DESCRIPTION from body
   - Key: use `normalize-tool-key` from tool-socket (or inline the same logic: `(intern (string-upcase (substitute #\- #\_ name)) :keyword)`)
   - Returns the hash-table

4. `reload-tool-definitions` function:
   - Calls `load-tool-definitions`, replaces `*tool-definition-cache*`
   - Prints count: `[tool-defs] Loaded N tool definitions from DB`
   - Wrapped in handler-case for error resilience

5. `lookup-tool-definition` function:
   - Takes tool-name string
   - Returns `(gethash (normalize-tool-key-local tool-name) *tool-definition-cache*)`
   - Also tries raw string key as fallback
   - Returns nil if not found

6. Local `normalize-tool-key-local` helper (same logic as tool-socket's normalize-tool-key to avoid circular dependency):
   - `(intern (string-upcase (substitute #\- #\_ name)) :keyword)`
