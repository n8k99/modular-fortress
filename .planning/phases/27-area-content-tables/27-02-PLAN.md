```lisp
(defun split-on-dot (s)
  "Split string S on '.' returning list of substrings."
  (let ((parts '()) (start 0))
    (loop for i from 0 below (length s)
          when (char= (char s i) #\.)
          do (push (subseq s start i) parts)
             (setf start (1+ i)))
    (push (subseq s start) parts)
    (nreverse parts)))
```

**Step 2: Modify load-bundle method** (currently at lines 314-332)

Replace the existing load-bundle method with one that FIRST checks for area.content pattern, then falls through to existing template lookup:

```lisp
(defmethod load-bundle ((r noosphere-resolver) name)
  "Load a {bundle_name} bundle. First checks for area.content pattern
(e.g., 'em.content' or 'em.content.podcast'), returning data plists directly.
Falls through to templates table for non-area bundles."
  ;; Check for area.content or area.content.type pattern
  (let ((parts (split-on-dot name)))
    (if (and (>= (length parts) 2)
             (string-equal (second parts) "content"))
        ;; Area content bundle -- return DATA, not AST nodes (per pitfall 2)
        (let* ((prefix (first parts))
               (area-slug (lookup-area-slug prefix))
               (content-type (when (>= (length parts) 3) (third parts))))
          (when area-slug
            (let ((results (resolve-from-area-content area-slug content-type)))
              (when results
                results))))  ;; Return list of plists directly
        ;; Default: template lookup (existing behavior, preserved exactly)
        (handler-case
            (let* ((esc (db-escape name))
                   (sql (format nil
                           "SELECT body FROM templates WHERE LOWER(name) = LOWER(~a) LIMIT 1"
                           esc))
                   (results (db-query sql)))
              (when (> (length results) 0)
                (let ((body (gethash :body (aref results 0))))
                  (when (and body (stringp body) (> (length body) 0))
                    (let* ((tokens (tokenize body))
                           (program (parse tokens)))
                      (node-children program))))))
          (error (e)
            (format t "  [load-bundle] error: ~a~%" e)
            nil)))))
```

Key design decisions per D-06, D-07, D-08:
- {em.content} -> resolve-from-area-content with area-slug="em-corp", no content-type filter
- {em.content.podcast} -> resolve-from-area-content with area-slug="em-corp", content-type="podcast"
- Returns list of plists (DATA), NOT AST nodes (per pitfall 2)
- Unknown area prefix (lookup-area-slug returns nil) -> returns nil (bundle not found)
- Template lookup preserved exactly as-is for non-area bundles
