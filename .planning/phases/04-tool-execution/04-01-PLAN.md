  <action>
    Create `/opt/project-noosphere-ghosts/tools/claude-code-tool.sh` following the tool-builder.sh pattern (per D-06):

    ```bash
    #!/bin/bash
    # claude-code-tool.sh — Execute coding tasks via Claude Code CLI
    # Called by tool-socket.lisp general script dispatch
    # Synchronous: returns result to stdout (unlike async tool-builder.sh)

    export HOME=/root
    export PATH="/root/.local/bin:/root/.nvm/versions/node/v22.22.0/bin:/usr/local/bin:/usr/bin:/bin"
    export TERM=dumb

    # Clean SBCL environment pollution (per research Pitfall 3)
    unset ANTHROPIC_API_KEY CLAUDE_API_KEY XDG_CONFIG_HOME XDG_DATA_HOME
    unset SBCL_HOME ASDF_OUTPUT_TRANSLATIONS

    PROMPT="$1"
    ALLOWED_TOOLS="${2:-Read,Grep,Glob,Write,Edit,Bash}"
    TIMEOUT=120

    if [ -z "$PROMPT" ]; then
        echo '{"error": "No prompt provided"}'
        exit 1
    fi

    timeout ${TIMEOUT}s claude -p "$PROMPT" \
      --allowedTools "$ALLOWED_TOOLS" \
      --model sonnet \
      --output-format json \
      2>/tmp/claude-code-tool-err.log

    EXIT_CODE=$?
    if [ $EXIT_CODE -ne 0 ]; then
        ERR=$(cat /tmp/claude-code-tool-err.log 2>/dev/null | tail -5)
        echo "{\"error\": \"Claude Code exited with code $EXIT_CODE\", \"stderr\": \"$ERR\"}"
    fi
    ```

    Make the script executable: `chmod +x /opt/project-noosphere-ghosts/tools/claude-code-tool.sh`

    Key differences from tool-builder.sh per research:
    - Synchronous (stdout captured by tool-socket) not async (uiop:launch-program)
    - 120s hard timeout via `timeout` command (per D-06)
    - No working directory change (uses current dir)
    - Returns JSON output directly to caller
    - Uses --model sonnet (cost-efficient for tool tasks)
