**Create `summarize.py`** (per D-07, D-08, D-13):
- Function `call_claude(prompt: str, max_retries: int = 3) -> str`:
  - `subprocess.run(['claude', '-p', prompt, '--output-format', 'json'], capture_output=True, text=True, timeout=180)`
  - Parse JSON response, extract `result` key
  - Retry with exponential backoff (2^attempt seconds) on failure
  - Return the result string

- Function `summarize_conversation(archive_id: int, title: str, date: str, content: str) -> dict`:
  - If content > 50000 chars, truncate to first 10000 + "\n\n[...truncated...]\n\n" + last 5000 (per research recommendation)
  - Format CONVERSATION_SUMMARY_PROMPT with date, title, truncated content
  - Call claude, parse the returned JSON string into dict with keys: summary, domains, key_topics
  - Return the parsed dict

- Function `run_summarize(dry_run=False, batch_size=20, start_from=0)`:
  1. Query non-trivial archives not yet summarized:
     ```sql
     SELECT a.id, a.title, a.period_start, a.content
     FROM archives a
     WHERE a.source_type = 'chatgpt_import'
       AND (a.metadata->>'trivial')::boolean = false
       AND a.metadata->>'summary' IS NULL
     ORDER BY a.period_start, a.id
     OFFSET %s
     ```
  2. For each conversation in batches of `batch_size`:
     - Call `summarize_conversation()`
     - UPDATE archive metadata: `UPDATE archives SET metadata = metadata || %s::jsonb WHERE id = %s` with `{"summary": ..., "domains": [...], "key_topics": [...]}`
     - Also UPDATE topic to first key_topic if topic is just the title: `UPDATE archives SET topic = %s WHERE id = %s AND topic = title`
     - Log: `"Summarized {i}/{total}: {title}"`
  3. Sleep 2 seconds between each call to respect rate limits
  4. Commit after each batch of `batch_size`
  5. Print final: `"Summarized {n} conversations. Skipped {s} already done."`

- CLI: argparse with `--dry-run`, `--batch-size N`, `--start-from N` flags.
