    echo "FAIL: $req - $desc"

**PERC-03: Staff perceives tasks with context and must_haves**
- First find a staff agent with GSD tasks. Use the assigned_to from dispatched tasks.
- Query: `psql -U chronicle -d master_chronicle -t -c "SELECT DISTINCT unnest(assigned_to) FROM tasks WHERE source='gsd' AND assigned_to IS NOT NULL LIMIT 1"`
- curl perception/{staff_agent}, jq for tasks with non-null context containing "must_haves"
- Test: `[.tasks[] | select(.context != null and (.context | contains("must_haves")))] | length > 0`
- If no staff agent has GSD tasks, test with eliana (executive also sees assigned GSD tasks)

- The actual +15 boost happens in Lisp tick-engine, not in the API. What we CAN verify:
  1. Eliana's perception returns non-empty `.projects` array (prerequisite for boost)
  2. The Lisp code at tick-engine.lisp contains `(* 15 (length projects))` (code exists)
- Test API: `.projects | length > 0` for eliana
- Test code: `grep -c "(\\* 15 (length projects))" /opt/project-noosphere-ghosts/lisp/runtime/tick-engine.lisp`
- Both must be true for PERC-04 to pass

**PERC-05: scheduled_at field present for Lisp client-side filtering**
- curl perception/eliana, jq for tasks that HAVE the scheduled_at key (even if null)
- Test: `.tasks[0] | has("scheduled_at")`
