Integrate mark-read into the ghost tick engine so that after cognition completes, the agent's processed messages are marked as read via the API. This closes the loop: perception filters read messages (Plan 01), and now the Lisp side actually marks them. Purpose: Without this integration, the mark-read endpoint exists but is never called, so ghosts would still re-process every message.

**Per D-01:** Messages are marked as read AFTER cognition completes, not during perception. Per Research open question #1, the cleanest approach is to pass `perceptions` into `phase-process-cognition`.

**Two changes required in tick-engine.lisp:**

**Change 1: Modify `phase-process-cognition` signature to accept `perceptions`**

Update the function definition (around line 318) from:
```lisp
(defun phase-process-cognition (tick-number drives agent-map agent-summaries)
```
to:
```lisp
(defun phase-process-cognition (tick-number drives agent-map agent-summaries perceptions)
```

**Change 2: Add mark-read call after successful `execute-cognition-result`**

Inside the `(handler-case (let ((action-detail (execute-cognition-result result))) (when action-detail ...))` block, AFTER the existing `(push (build-log-entry ...) logs)` call (around line 357) and still inside the `(when action-detail ...)` form, add:

```lisp
;; Mark processed messages as read (SPAM-02)
(let* ((agent-id-for-read (cognition-result-agent-id result))
       (perception (when perceptions (gethash agent-id-for-read perceptions)))
       (msgs (when perception (gethash :messages perception)))
       (msg-ids (when (and msgs (> (length msgs) 0))
                  (coerce (map 'vector (lambda (m) (gethash :id m)) msgs) 'vector))))
  (when (and msg-ids (> (length msg-ids) 0))
    (handler-case
        (api-post "/api/conversations/mark-read"
                  (json-object :agent-id agent-id-for-read
                               :message-ids msg-ids))
      (error (e)
        (format t "  [mark-read-error] ~a: ~a~%" agent-id-for-read e)))))
```

This extracts ALL message IDs from the agent's perception snapshot (per Research pitfall #4 recommendation: mark all perceived messages as read after successful cognition, not just the specific responded-to message).

The `handler-case` wrapping ensures mark-read failures are logged but do not crash the tick engine or prevent other agents from processing.

Note: The variable is named `agent-id-for-read` to avoid shadowing the existing `agent-id` binding in the outer let*.

**Change 3: Update the call site in `run-tick`**

In `run-tick` (around line 487-488), update the call from:
```lisp
(phase-process-cognition tick-number drives agent-map agent-summaries)
```
to:
```lisp
(phase-process-cognition tick-number drives agent-map agent-summaries perceptions)
```

The `perceptions` variable is already in scope in `run-tick` (set at line 471).
