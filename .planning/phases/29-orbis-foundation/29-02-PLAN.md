Add a convenience function `load-ghost-orbis` to ghost-capabilities.lisp that reads the Orbis-specific fields from a ghost's YAML. This satisfies ORBIS-03's "readable at runtime" requirement (per D-10, readable but not enforced).

```lisp
(defun load-ghost-orbis (agent-id)
  "Read YAML file for AGENT-ID and return a plist of Orbis fields:
   (:starting-point ht :ship-assignment string :rpg-persona ht :orbis-access ht)
   Returns NIL if file does not exist or has no Orbis fields."
  (handler-case
      (let ((path (ghost-yaml-path agent-id)))
        (when (probe-file path)
          (let* ((content (uiop:read-file-string path))
                 (ht (af64.utils.yaml:parse-simple-yaml content))
                 (sp (gethash "starting_point" ht))
                 (sa (gethash "ship_assignment" ht))
                 (rp (gethash "rpg_persona" ht))
                 (oa (gethash "orbis_access" ht)))
            (when (or sp sa rp oa)
              (list :starting-point sp
                    :ship-assignment sa
                    :rpg-persona rp
                    :orbis-access oa)))))
    (error (e)
      (format t "  [ghost-orbis] Error loading orbis for ~a: ~a~%" agent-id e)
      nil)))
```

Then verify by calling load-ghost-orbis for all 9 agents and confirming non-nil results with correct types.
