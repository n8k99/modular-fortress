1. Add a helper function `format-project-tasks` BEFORE `build-project-review-job` in action-planner.lisp. This function:
   - Takes a project ID (integer)
   - Calls `(api-get "/api/af64/tasks" (list :project-id project-id :status "open" :limit 50))`
   - For each returned task, formats: task ID, text, status, assignee, and parses the `context` field (JSON string) to extract wave number and must_haves
   - Returns a formatted string with task details grouped by wave
   - Wraps in `handler-case` to return empty string on API errors
   - Limits context JSON to key fields only (wave, must_haves summary) to avoid prompt bloat (per Pitfall 5)

   Pattern:
   ```lisp
   (defun format-project-tasks (project-id)
     "Fetch and format tasks for a project, grouped by wave for executive review."
     (handler-case
         (let* ((tasks (api-get "/api/af64/tasks"
                                (list :project-id project-id :status "open" :limit 50)))
                (task-list (if (vectorp tasks) (coerce tasks 'list)
                              (if (listp tasks) tasks nil))))
           (if (null task-list)
               "  No open tasks."
               (with-output-to-string (s)
                 (dolist (task task-list)
                   (let* ((tid (gethash :id task))
                          (text (or (gethash :text task) ""))
                          (assignee (or (gethash :assignee task) "unassigned"))
                          (ctx-raw (gethash :context task))
                          (ctx (when (and ctx-raw (stringp ctx-raw) (> (length ctx-raw) 2))
                                 (handler-case (parse-json ctx-raw) (error () nil))))
                          (wave (when (and ctx (hash-table-p ctx)) (gethash :wave ctx))))
                     (format s "  - Task #~a [wave ~a] (~a): ~a~%"
                             tid (or wave "?") assignee
                             (subseq text 0 (min 120 (length text)))))))))
       (error (e)
         (format nil "  [error loading tasks: ~a]" e))))
   ```
