**Section 3: Generate and execute YAML frontmatter injection** (per D-11):

The YAML frontmatter must be prepended to each document's content. Build the UPDATE statements dynamically by querying the current state of agents, teams, team_members, departments, and areas.

For each of the 64 agents, generate:
```sql
UPDATE documents SET content =
  '---' || E'\n' ||
  'agent_id: ' || a.id || E'\n' ||
  'memory_column: ' || COALESCE(a.id || '_memories', '~') || E'\n' ||
  'department: ' || dep.name || E'\n' ||
  'team: ' || COALESCE(t.name, 'Unassigned') || E'\n' ||
  'area: ' || COALESCE(ar.name, 'Unassigned') || E'\n' ||
  '---' || E'\n\n' ||
  d.content
FROM agents a
JOIN documents d ON d.id = a.document_id
LEFT JOIN departments dep ON dep.id = a.department_id
LEFT JOIN team_members tm ON tm.agent_id = a.id
LEFT JOIN teams t ON t.id = tm.team_id
LEFT JOIN agent_areas aa ON aa.agent_id = a.id AND aa.role = 'primary'
LEFT JOIN areas ar ON ar.id = aa.area_id
WHERE documents.id = a.document_id
  AND d.path LIKE 'Areas/Eckenrode Muziekopname/EM Staff/%';
```

CRITICAL CONSIDERATIONS:
- **Nathan exception** (research Pitfall 2): Nathan has no memory column. His YAML should have `memory_column: ~` (YAML null). Check: `CASE WHEN a.id = 'nathan' THEN '~' ELSE a.id || '_memories' END`.
- **Duplicate team memberships**: If an agent is in multiple teams (e.g., an exec in both Executive team and their dept team), pick the non-Executive team for the `team:` frontmatter field (their operational team). Use: `WHERE t.name <> 'Executive' OR NOT EXISTS (SELECT 1 FROM team_members tm2 JOIN teams t2 ON t2.id = tm2.team_id WHERE tm2.agent_id = a.id AND t2.name <> 'Executive')`.
- **No existing frontmatter** (research Pitfall 5): Research confirms no EM Staff docs have `---` frontmatter yet. Verify with: `SELECT COUNT(*) FROM documents WHERE path LIKE 'Areas/Eckenrode Muziekopname/EM Staff/%' AND content LIKE '---\n%'` should return 0 before injection.
- **Content preservation**: The UPDATE must ONLY prepend — never modify existing content. Use string concatenation: `'---\n...\n---\n\n' || content`.

If the single UPDATE with JOINs is too complex to get right for all 64 agents at once, generate individual UPDATE statements per agent. Use a DO block or generate SQL from a query:

```sql
SELECT format(
  $$UPDATE documents SET content = '---' || E'\nagent_id: %s\nmemory_column: %s\ndepartment: %s\nteam: %s\narea: %s\n---\n\n' || content WHERE id = %s;$$,

LEFT JOIN LATERAL (
  SELECT ar2.name FROM agent_areas aa2
  JOIN areas ar2 ON ar2.id = aa2.area_id
  WHERE aa2.agent_id = a.id AND aa2.role = 'primary'
  LIMIT 1
) ar ON true
WHERE a.document_id IS NOT NULL;
```
