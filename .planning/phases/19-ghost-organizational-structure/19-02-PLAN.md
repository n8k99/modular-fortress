**Section 2: Populate team_members** from live agent data.
Query each agent's department_id and the `Team:` field from their EM Staff document to determine team membership. Use this pattern:
```sql
-- Extract team name from EM Staff doc content and join to teams table
INSERT INTO team_members (team_id, agent_id, role_in_team)
SELECT t.id, a.id,
  CASE
    WHEN a.id = t.lead_id THEN 'lead'
    WHEN a.role ILIKE '%executive%' OR a.role ILIKE '%chief%' OR a.role ILIKE '%head%' THEN 'executive'
    ELSE 'member'
  END
FROM agents a
JOIN documents d ON d.id = a.document_id
JOIN teams t ON d.content LIKE '%**Team:** ' || t.name || '%'
WHERE a.document_id IS NOT NULL;
```

**Section 3: Migrate relationships to ghost_relationships** (per D-05, using research SQL patterns):

```sql
-- reports_to: text[] -> rows
INSERT INTO ghost_relationships (from_agent, to_agent, relationship_type)
SELECT a.id, unnest(a.reports_to::text[]), 'reports_to'
FROM agents a
WHERE a.reports_to IS NOT NULL AND array_length(a.reports_to::text[], 1) > 0;

-- mentor: scalar text -> single row
INSERT INTO ghost_relationships (from_agent, to_agent, relationship_type)
SELECT a.id, a.mentor, 'mentor'
FROM agents a
WHERE a.mentor IS NOT NULL AND a.mentor <> '';

-- mentee: derive from inverse of mentor (per research: mentee column is empty)
INSERT INTO ghost_relationships (from_agent, to_agent, relationship_type)
SELECT a.mentor, a.id, 'mentee'
FROM agents a
WHERE a.mentor IS NOT NULL AND a.mentor <> '';

-- collaborators: text[] -> rows
INSERT INTO ghost_relationships (from_agent, to_agent, relationship_type)
SELECT a.id, unnest(a.collaborators::text[]), 'collaborators'
FROM agents a
WHERE a.collaborators IS NOT NULL AND array_length(a.collaborators::text[], 1) > 0;

-- liaises_with: text[] -> rows
INSERT INTO ghost_relationships (from_agent, to_agent, relationship_type)
SELECT a.id, unnest(a.liaises_with::text[]), 'liaises_with'
FROM agents a
WHERE a.liaises_with IS NOT NULL AND array_length(a.liaises_with::text[], 1) > 0;
```

NOTE: Check the actual column types first. If reports_to, collaborators, liaises_with are already text[] arrays, no cast needed. If they are plain text, parse accordingly. The research says they are text[] (array columns).
