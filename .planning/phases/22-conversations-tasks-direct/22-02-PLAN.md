**Transformation rules (apply to each call site):**

1. **api-post "/api/conversations" (13 sites)** -> Per D-01:
```lisp
;; BEFORE:
```

2. **api-patch "/api/af64/tasks/:id" (12 sites)** -> Per D-04:
```lisp

;; AFTER:

```
For task completion sites that also need unblocking (look for patterns where status='done' AND there's subsequent blocked_by logic), use db-complete-and-unblock instead:
```lisp
(db-complete-and-unblock task-id :stage-notes notes)
```

3. **api-get "/api/af64/tasks..." (4 sites)** -> Per D-06:

For single task fetch by ID: `(db-get-task-by-id task-id)`
