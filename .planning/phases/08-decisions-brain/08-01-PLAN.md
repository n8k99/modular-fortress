**Step 2: Create `/opt/dpn-api/src/handlers/decisions.rs`** following the af64_tasks.rs handler pattern exactly.

Define two structs:
- `DecisionQuery` (Deserialize): project_id (Option<i32>), department (Option<String>), owner (Option<String>), limit (Option<i64>), order (Option<String> -- "asc" or "desc", default "desc")
- `NewDecision` (Deserialize): decision (String, required), rationale (Option<String>), project_id (Option<i32>), department (Option<String>), owner (String, required), stakeholders (Option<serde_json::Value>), date (Option<String> -- ISO date, defaults to today)

Per D-05, implement `list_decisions`:
- Accept `Query<DecisionQuery>` and `State<DbPool>`
- Default limit to 20, max 100
- Default order to "desc" (most recent first per D-04)
- Build SQL dynamically based on which filters are present:
  - If project_id set: `WHERE project_id = $1`
  - Else if department set: `WHERE department = $1`
  - Else if owner set: `WHERE owner = $1`
  - Else: no WHERE clause
- ORDER BY created_at {ASC|DESC} LIMIT $N
- SELECT columns: id, project_id, decision, rationale, owner, stakeholders, department, date, created_at
- Map rows to JSON using `r.get::<Type, _>("column")` pattern (same as af64_tasks.rs)
- Use `Option<chrono::NaiveDate>` for date, `Option<chrono::DateTime<chrono::Utc>>` for created_at, `Option<serde_json::Value>` for stakeholders

Per D-06, implement `create_decision`:
- Accept `Json<NewDecision>` and `State<DbPool>`
- Parse date from body.date using `chrono::NaiveDate::parse_from_str(d, "%Y-%m-%d")`, default to `chrono::Utc::now().date_naive()`
- INSERT INTO decisions (decision, rationale, project_id, department, owner, stakeholders, date) VALUES ($1-$7) RETURNING id, created_at
- Return JSON with id and created_at

Per D-07, do NOT add PUT or DELETE endpoints. Decisions are append-only.

Use `format!()` for dynamic ORDER BY clause (cannot bind ORDER BY direction as parameter). This is safe because the value is validated to be "ASC" or "DESC" only.

IMPORTANT: Follow the Rust UTF-8 rule from CLAUDE.md -- never byte-slice decision text. Use `.chars().take(N).collect::<String>()` if any truncation is needed.
