```lisp
(defun extract-innate-expressions (content)
  "Extract innate_expressions JSON array from LLM output content.
   Looks for \"innate_expressions\" key in JSON. Returns list of hash-tables or NIL."
  (handler-case
      (let ((key-pos (search "\"innate_expressions\"" content)))
        (when key-pos
          ;; Find the opening [ of the array value
          (let ((arr-start (position #\[ content :start key-pos)))
            (when arr-start
              ;; Find matching closing ]
              (let ((depth 1) (pos (1+ arr-start)))
                (loop while (and (< pos (length content)) (> depth 0))
                      do (let ((ch (char content pos)))
                           (cond ((char= ch #\[) (incf depth))
                                 ((char= ch #\]) (decf depth)))
                           (incf pos)))
                (when (= depth 0)
                  (let* ((json-str (subseq content arr-start pos))
                         (parsed (parse-json json-str)))
                    (when (vectorp parsed)
                      (coerce parsed 'list)))))))))
    (error () nil)))
```

2. **Add `process-innate-expressions` function:**

```lisp
(defun process-innate-expressions (content agent-id)
  "Extract, validate, and persist innate expressions from LLM output.
   Returns count of successfully persisted expressions."
  (let ((expressions (extract-innate-expressions content))
        (count 0))
    (when expressions
      (format t "  [innate-gen] ~a: ~a expression(s) found~%" agent-id (length expressions))
      (dolist (expr-def expressions)
        (when (hash-table-p expr-def)
          (let* ((name (or (gethash :name expr-def) (gethash :NAME expr-def)))
                 (body (or (gethash :body expr-def) (gethash :BODY expr-def)))
                 (category (or (gethash :category expr-def) (gethash :CATEGORY expr-def) "ghost-generated"))
                 (description (or (gethash :description expr-def) (gethash :DESCRIPTION expr-def)))
                 (update-name (or (gethash :update expr-def) (gethash :UPDATE expr-def))))
            (cond
              ;; CREATE path: name + body
              ((and name body (not update-name))
               (multiple-value-bind (valid err)
                   (validate-innate-expression body)
                 (if valid
                     (multiple-value-bind (ok insert-err)
                         (db-insert-template name body :category category :description description)
                       (if ok
                           (progn (incf count)
                                  (format t "  [innate-gen] ~a: created template '~a'~%" agent-id name))
                           (format t "  [innate-gen-error] ~a: INSERT failed for '~a': ~a~%" agent-id name insert-err)))
                     (format t "  [innate-gen-invalid] ~a: expression for '~a' failed validation: ~a~%" agent-id name err))))
              ;; UPDATE path: update (name to look up) + body
              ((and update-name body)
               (multiple-value-bind (valid err)
                   (validate-innate-expression body)
                 (if valid
                     (let ((existing (db-find-template-by-name update-name)))
                       (if existing
                           (let ((tid (gethash :ID existing)))
                             (multiple-value-bind (ok upd-err)
                                 (db-update-template-body tid body)
                               (if ok
                                   (progn (incf count)
                                          (format t "  [innate-gen] ~a: updated template '~a' (id=~a)~%" agent-id update-name tid))
                                   (format t "  [innate-gen-error] ~a: UPDATE failed for '~a': ~a~%" agent-id update-name upd-err))))
                           (format t "  [innate-gen-error] ~a: template '~a' not found for update~%" agent-id update-name)))
                     (format t "  [innate-gen-invalid] ~a: update expression for '~a' failed validation: ~a~%" agent-id update-name err))))
              ;; Missing required fields
              (t
               (format t "  [innate-gen-skip] ~a: expression missing name or body~%" agent-id)))))))
    count))
```

When your task involves creating or modifying Templates, you can generate Innate .dpn expressions.
Include them in your response as a JSON block with the key \"innate_expressions\".

### Syntax Reference
- @reference: `@project_name` or `@projects{status=active}` -- references an entity
- (agent){action}: `(nova){health_check}` -- commissions an agent to perform an action
- ![search]: `![projects]{status=blocked}` -- searches for entities matching criteria

To CREATE a new template:
```json
{\"innate_expressions\": [{\"name\": \"Template Name\", \"body\": \"@projects{status=active}\\n(nova){health_check}\", \"category\": \"ghost-generated\", \"description\": \"What this template does\"}]}
```

To UPDATE an existing template:
```json
{\"innate_expressions\": [{\"update\": \"Existing Template Name\", \"body\": \"new body content here\"}]}
```

### Rules
- Expression bodies MUST be valid Innate syntax (they are validated by the parser before persistence)
- Agent IDs use DB id column: nova, eliana, sarah, kathryn, sylvia, vincent, jmax, lrm
- Only include innate_expressions when your task explicitly involves template creation or modification
- If not creating/modifying templates, do NOT include innate_expressions in your response
")
