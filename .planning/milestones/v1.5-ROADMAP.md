## Milestone Summary

**Key Decisions:**
- SB-ALIEN FFI to libpq (zero-deps, no Quicklisp) -- proven from v1.4
- CLOS resolver protocol for Innate -- extensible method dispatch
- LLM-generated expressions with parse-round-trip validation
- YAML-first capability injection: ghost YAML authoritative, tool-registry fallback removed
- Pipeline definitions in area_content JSONB (DB is the OS)
- Tool metadata in area_content JSONB (75 tools migrated)
- Flat cond in YAML parser to avoid deep paren nesting
- Custom YAML parser (zero-deps) for agent config

**Issues Resolved:**
- execute-work-task paren scope bug (STAB-01) -- surgical 2-char fix
- 7 tick engine runtime errors (UTF-8, NULL, tilde SQL, type coercion, description column)
- tool-registry.json hallucination problem -- ghosts no longer guess tool names
- Stage name collision ("research" in both editorial and investment pipelines) fixed by (pipeline, stage) compound key

**Issues Deferred:**
- Join logic for pipeline convergence (forks supported, joins deferred)
- Parent-child project relationships as schema column
- af64.asd out of sync with runtime (launch.sh is actual loader)
- Orbis runtime enforcement (load-ghost-orbis exported but uncalled until v1.6)

**Technical Debt:**
- tick-engine.lisp double-loaded in launch.sh (harmless, wasteful)
- validate-stage-output has hardcoded stage list
- TOOL-04 uses "resolver" as sender attribution, not executing ghost agent_id
- Nyquist validation partial across all phases (no automated test framework for Lisp runtime)
