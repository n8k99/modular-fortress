**CRITICAL PATTERNS (per D-17):**
- Use `db-query-single` for INSERT...RETURNING (returns hash-table). NEVER use `db-execute` for RETURNING queries (Pitfall 1 from research).
- Use `db-execute` for UPDATE/DELETE without RETURNING.
- Use `db-escape` for ALL string values. Use unescaped integers directly.
- PostgreSQL array literals: use ARRAY[escaped1,escaped2]::type[] constructor syntax, never '{a,b}' string literal syntax.
- Results come back as hash-tables with hyphenated keywords (`:from-agent` not `:from_agent`).
- handler-case per SQL call for error isolation.

**Helper functions (put in db-auxiliary.lisp, export from there):**
```lisp
(defun pg-text-array (items)
  "Build ARRAY['a','b']::varchar(50)[] from list of strings. Empty list returns '{}'::varchar(50)[]."
  ...)
(defun pg-int-array (items)
  "Build ARRAY[1,2]::integer[] from list of integers. Empty list returns '{}'::integer[]."
  ...)
```

**db-conversations.lisp (per D-01, D-02, D-03):**

1. `db-insert-conversation (from-agent to-agents message &key channel thread-id metadata)` -- INSERT INTO conversations with RETURNING id. to-agents is a Lisp list of strings; use pg-text-array. thread-id: if provided, escape it; if nil, use gen_random_uuid(). metadata: if hash-table, encode-json + escape; if nil, use '{}'::jsonb. Per D-01. Use db-query-single (not db-execute) because RETURNING.

2. `db-mark-read-batch (agent-id message-ids)` -- Per D-02. For each message-id in message-ids: `UPDATE conversations SET read_by = array_append(read_by, agent_str) WHERE id = msg_id AND NOT (agent_str = ANY(read_by))`. IMPORTANT: read_by is varchar(50)[], NOT integer[] (Pitfall 3). agent-id is a string like "nova". Use db-execute for each UPDATE (no RETURNING needed).

3. `db-get-conversations (&key thread-id agent-id unread-for limit)` -- Per D-03. SELECT id, thread_id, from_agent, to_agent, message, channel, message_type, metadata, read_by, created_at FROM conversations WHERE ... ORDER BY created_at DESC. If unread-for provided: `AND NOT (agent_str = ANY(read_by))`. If thread-id: `AND thread_id = escaped_thread`. If agent-id: `AND (from_agent = escaped_agent OR escaped_agent = ANY(to_agent))`. LIMIT defaults to 50. Use db-query.

**db-tasks.lisp (per D-04, D-05, D-06, D-07):**

1. `db-update-task (task-id &key status stage-notes output-artifact completed-date blocked-by assigned-to stage)` -- Per D-04. Build dynamic SET clause from provided keyword args only. stage-notes and output-artifact are jsonb (encode-json + escape + ::jsonb cast). blocked-by uses pg-int-array. assigned-to uses pg-text-array. Use db-execute (no RETURNING).

2. `db-create-task (text &key project-id assigned-to priority status goal-id blocked-by department description)` -- Per D-05. INSERT INTO tasks (task_id, text, status, assignee, assigned_to, department, doc_path, line_number, raw_line, project_id, goal_id, blocked_by, source, description) VALUES (...) RETURNING id, task_id. task_id = format "ghost-~a" with generate-uuid. doc_path = 'af64/generated'. line_number = 0. raw_line = text. source = 'ghost'. status defaults to 'open'. Use db-query-single.

3. `db-get-tasks-by-filter (&key assigned-to status goal-id limit task-id)` -- Per D-06. SELECT id, task_id, text, status, assignee, assigned_to, department, project_id, goal_id, blocked_by, stage_notes, stage, source, doc_path, scheduled_at, deadline, recurrence, description FROM tasks WHERE ... LIMIT. If assigned-to: `AND (assignee = esc OR esc = ANY(assigned_to))`. If status: `AND status = esc`. If goal-id: `AND goal_id = int`. If task-id: `AND id = int`. Default limit 50. Use db-query.

4. `db-get-task-by-id (task-id)` -- Convenience: SELECT * FROM tasks WHERE id = task-id. Use db-query-single.

5. `db-complete-and-unblock (task-id &key stage-notes completed-date)` -- Per D-07. Transaction: BEGIN; UPDATE tasks SET status='done', completed_date=escaped_date, stage_notes=escaped_notes WHERE id=task-id; UPDATE tasks SET blocked_by = array_remove(blocked_by, task-id) WHERE task-id = ANY(blocked_by); COMMIT; Use db-execute (multi-statement, no RETURNING needed).
