**Helper function: hash-to-plist**
Convert db-query hash-table results to plists. Skip NULL (nil) values. Keys are already hyphenated keywords from AF64 JSON parser.

```common-lisp
(defun hash-to-plist (ht)
  "Convert hash-table with keyword keys to plist, skipping nil values."
  (when ht
    (let ((plist '()))
      (maphash (lambda (k v) (when v (push v plist) (push k plist))) ht)
      plist)))
```

**Entity resolution helpers (per D-04, D-05, D-06):**

Create `resolve-from-TABLE` functions for each of agents, projects, areas, templates, resources. Each:
1. Takes a name string
2. Runs case-insensitive SQL query via db-query + db-escape
3. Returns plist or nil

SQL patterns (use db-escape for the name parameter):
- **agents**: `SELECT id, full_name, role, department, status, agent_tier, description, reports_to FROM agents LEFT JOIN agent_state ON agents.id = agent_state.agent_id WHERE LOWER(id) = LOWER({esc}) OR LOWER(full_name) = LOWER({esc}) LIMIT 1` -- JOIN agent_state to get energy and tier per research open question 2
- **projects**: `SELECT id, name, status, owner, description, goals, lifestage FROM projects WHERE LOWER(name) = LOWER({esc}) LIMIT 1` -- use `name` column NOT `title` per Pitfall 2
- **areas**: `SELECT id, name, slug, description, owner, status FROM areas WHERE LOWER(name) = LOWER({esc}) LIMIT 1`
- **templates**: `SELECT id, name, slug, category, description, body, parameters FROM templates WHERE LOWER(name) = LOWER({esc}) LIMIT 1`
- **resources**: `SELECT id, name, slug, resource_type, description, tags FROM resources WHERE LOWER(name) = LOWER({esc}) LIMIT 1`

Each helper wraps in `handler-case` returning nil on error (log with format t).

Use `(aref results 0)` not `(first results)` since db-query returns vectors (Pitfall 3).

**Table-dot-name dispatch (per D-05):**
```common-lisp
(defun resolve-from-table (table-name entity-name)
  "Dispatch to specific table resolver. TABLE-NAME is pluralized string."
  (let ((tbl (string-downcase table-name)))
    (cond
      ((string= tbl "agents") (resolve-from-agents entity-name))
      ((string= tbl "projects") (resolve-from-projects entity-name))
      ((string= tbl "areas") (resolve-from-areas entity-name))
      ((string= tbl "templates") (resolve-from-templates entity-name))
      ((string= tbl "resources") (resolve-from-resources entity-name))
      (t nil))))
```

**resolve-reference method (per D-04, D-05, D-06, D-07, D-16, D-17):**
```common-lisp
(defmethod resolve-reference ((r noosphere-resolver) name qualifiers)
  ;; Check for table.name dispatch (D-05)
  (let ((dot-pos (position #\. name)))
    (if dot-pos
        ;; Table-specific lookup
        (let* ((table (subseq name 0 dot-pos))
               (entity-name (subseq name (1+ dot-pos)))
               (entity (resolve-from-table table entity-name)))
          (handle-entity-result entity entity-name qualifiers))
        ;; Cascade: agents -> projects -> areas -> templates -> resources (D-04)
        (let ((entity (or (resolve-from-agents name)
                          (resolve-from-projects name)
                          (resolve-from-areas name)
                          (resolve-from-templates name)
                          (resolve-from-resources name))))
          (handle-entity-result entity name qualifiers)))))
```

**handle-entity-result helper:**
If entity is nil, return `(make-resistance :message "Entity not found: {name}" :source name)` (D-16).
If qualifiers is nil, return `(make-innate-result :value entity :context :query)`.
If qualifiers exist, intern first qualifier as keyword via `(intern (string-upcase qual) :keyword)`, use `(getf entity key)`. If val found return innate-result, else return resistance with "has no property" message (D-07).

```common-lisp
(defun search-type-to-table (search-type)
  "Map Innate search type string to PostgreSQL table name."
  (let ((st (string-downcase (if (stringp search-type) search-type (format nil "~a" search-type)))))
    (cond

```
