**2. FFI bindings in pg.lisp** — Load libpq and define alien routines:

Define these alien routines (all use `(* t)` for opaque PG pointers):
- `PQconnectdb` — `(conninfo c-string) -> (* t)`
- `PQstatus` — `(conn (* t)) -> int` (0 = CONNECTION_OK)
- `PQexec` — `(conn (* t)) (query c-string) -> (* t)`
- `PQresultStatus` — `(res (* t)) -> int` (1=COMMAND_OK, 2=TUPLES_OK)
- `PQntuples` — `(res (* t)) -> int`
- `PQnfields` — `(res (* t)) -> int`
- `PQgetvalue` — `(res (* t)) (row int) (col int) -> c-string`
- `PQfname` — `(res (* t)) (col int) -> c-string`
- `PQgetisnull` — `(res (* t)) (row int) (col int) -> int` (1 = null)
- `PQclear` — `(res (* t)) -> void`
- `PQfinish` — `(conn (* t)) -> void`
- `PQescapeLiteral` — `(conn (* t)) (str c-string) (length sb-alien:unsigned-long) -> c-string`
  NOTE: PQescapeLiteral returns a malloc'd string. Define `PQfreemem` to free it:
- `PQfreemem` — `(ptr (* t)) -> void`
- `PQerrorMessage` — `(conn (* t)) -> c-string`

**3. Connection pool struct:**
```lisp
(defstruct pg-pool
  (connections (make-array 2 :initial-element nil))
  (in-use (make-array 2 :initial-element nil))
  (conninfo ""))
```

**4. Pool functions:**
- `(pool-acquire pool)` — iterate connections array, find one where `(aref in-use i)` is nil. If connection is nil or `(pqstatus conn)` is not 0, call `(pqfinish conn)` if non-nil, then `(pqconnectdb conninfo)` and check status. Set `(aref in-use i)` to t. Return `(values conn i)`. If no free connections, `(error "No available PG connections in pool")`.
- `(pool-release pool index)` — set `(aref in-use index)` to nil.
- `(pool-shutdown pool)` — iterate all connections, call PQfinish on non-nil ones, set to nil.

**5. Escape function:**
- `(pg-escape pool str)` — acquire a connection, call PQescapeLiteral (pass string and its length), copy result to Lisp string, call PQfreemem on the C string, release connection. Return the escaped string (which includes surrounding single quotes). If str is nil, return "NULL" (unquoted). If PQescapeLiteral returns null pointer, error with PQerrorMessage.

**6. Result conversion:**
- `(result-to-vectors res)` — Convert PG result to vector of hash-tables. For each row, create a hash-table (:test #'equal). For each column: get column name via PQfname, convert to keyword via `(json-keyword name)` (this does underscore-to-hyphen). Check PQgetisnull; if 1, set value to :null. Otherwise get string value via PQgetvalue and set it. Return a vector of hash-tables.

**7. Type-aware result conversion helper:**
- `(pg-coerce-value raw-value col-name)` — For now, return the raw string. Type coercion will be handled at the db-client layer where we know the query context. This keeps pg.lisp generic.

**8. Health check:**
- `(pg-health-check pool)` — acquire connection, execute "SELECT 1", check result status = TUPLES_OK (2), clear result, release connection. Return t on success, signal error on failure.

**9. Low-level query/execute:**
- `(pg-query pool sql)` — acquire connection, PQexec, check status = TUPLES_OK (2), call result-to-vectors, PQclear in unwind-protect, release in unwind-protect. Return vector of hash-tables.
- `(pg-execute pool sql)` — acquire connection, PQexec, check status = COMMAND_OK (1), PQclear in unwind-protect, release in unwind-protect. Return t.

CRITICAL: Every PQexec result MUST be wrapped in unwind-protect with PQclear. Every pool-acquire MUST be wrapped in unwind-protect with pool-release.

**`init-db-pool`** — Create pool using build-conninfo with D-05 defaults (host=127.0.0.1, port=5432, dbname=master_chronicle, user=chronicle, password=chronicle2026). Allow overriding via environment variables `PG_HOST`, `PG_PORT`, `PG_DB`, `PG_USER`, `PG_PASSWORD`. Call pg-health-check after creation to fail fast. Print status message. Set `*db-pool*`.

**`shutdown-db-pool`** — Call pool-shutdown on *db-pool*, set to nil.

**`db-query`** — Wrapper around pg-query that takes *db-pool* and SQL string. Returns vector of hash-tables.

**`db-execute`** — Wrapper around pg-execute that takes *db-pool* and SQL string. Returns t.

**`db-escape`** — Wrapper around pg-escape that takes *db-pool* and a value. Handle type dispatch:
- nil -> returns "NULL" (no quotes)
- string -> call pg-escape (returns single-quoted escaped string)
- integer -> `(format nil "~d" value)` (no quotes needed)
- float -> `(format nil "~f" value)` (no quotes needed)
- keyword -> call pg-escape on `(keyword->json-key value)` (convert to JSON-style string, then escape)
- hash-table -> call pg-escape on `(encode-json value)` (serialize to JSON, then escape as string for JSONB columns)

**`db-query-single`** — `(db-query-single sql)` — run db-query, return first row or nil.

**`db-coerce-row`** — `(db-coerce-row row type-map)` — Given a hash-table row and a type-map (alist of keyword -> type), coerce string values:
- `:integer` -> `(parse-integer value :junk-allowed t)`
- `:float` -> `(read-from-string value)` but only if value is a string
- `:boolean` -> `(string-equal value "t")`
- `:json` -> `(parse-json value)` if value is a string
- `:timestamp` -> keep as string (existing code handles ISO timestamps as strings)
- `:null-to-nil` -> if value is `:null`, set to nil
Return the modified row (mutates in place).
