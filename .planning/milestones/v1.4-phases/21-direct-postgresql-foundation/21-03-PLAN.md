**1. Add state update functions to db-client.lisp:**

**Function: `db-update-energy`** `(pool agent-id delta)` -> integer (new energy value)
```sql
UPDATE agent_state SET energy = GREATEST(0, LEAST(100, energy + {delta}))
WHERE agent_id = {escaped-id} RETURNING energy
```
Use db-query (not db-execute) because we need the RETURNING value. Parse the energy value from the first row as an integer/float. If no rows returned (agent not in agent_state), return 50 (the starting energy constant).

**Function: `db-get-energy`** `(pool agent-id)` -> integer
```sql
SELECT energy FROM agent_state WHERE agent_id = {escaped-id}
```
Parse energy as a number. If no rows, return 50.

**Function: `db-update-agent-state`** `(pool agent-id &key tier ticks-alive ticks-at-current-tier metadata-json)` -> t
Build a single UPDATE statement that sets all provided fields in one query:
```sql
UPDATE agent_state SET
  tier = {escaped-tier},
  last_tick_at = now(),
  ticks_alive = {ticks-alive},
  ticks_at_current_tier = {ticks-at-current-tier},
  metadata = COALESCE(metadata, '{}'::jsonb) || {escaped-metadata}::jsonb
WHERE agent_id = {escaped-id}
```
Where:
- `{escaped-tier}` = `(db-escape pool tier)`
- `{ticks-alive}` = integer, format directly
- `{ticks-at-current-tier}` = integer, format directly
- `{escaped-metadata}` = `(db-escape pool (encode-json metadata-json))` — the metadata is a hash-table, encode to JSON string then escape
- `{escaped-id}` = `(db-escape pool agent-id)`

Per af64_agents.rs behavior: last_tick_at is always set to now(). Metadata is merged (||) with existing, not replaced.

**Function: `db-set-energy`** `(pool agent-id energy)` -> t
For absolute energy setting (used in boss-override and some waking scenarios):
```sql
UPDATE agent_state SET energy = GREATEST(0, LEAST(100, {energy}))
WHERE agent_id = {escaped-id}
```
