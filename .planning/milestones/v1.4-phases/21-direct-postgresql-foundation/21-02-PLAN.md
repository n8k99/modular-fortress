Add the following functions to db-client.lisp. These implement the full perception query set from af64_perception.rs, restructured as separate queries per D-06. All SQL string values MUST use db-escape. All array columns MUST use `array_to_json()` in SQL. All JSONB columns MUST be cast to `::text` and then parsed with parse-json.

**Function: `db-perceive-messages`** `(pool agent-id since)` -> vector of hash-tables
SQL (derived from af64_perception.rs lines 33-56):
```sql
SELECT id, from_agent, LEFT(message, 500) as message, channel,
       thread_id::text, metadata->>'source' as source
FROM conversations
WHERE ({escaped-id} = ANY(to_agent) OR message ILIKE '%@' || {agent-id-unquoted} || '%')
  AND from_agent != {escaped-id}
  AND NOT ({escaped-id} = ANY(read_by))
  AND (created_at > {escaped-since}::timestamptz
       OR (metadata->>'source' = 'handoff'
           AND NOT EXISTS (
             SELECT 1 FROM conversations r
             WHERE r.from_agent = {escaped-id}
               AND r.metadata->>'responding_to' = conversations.id::text)))
ORDER BY CASE WHEN metadata->>'source' = 'handoff' THEN 0 ELSE 1 END,
         created_at DESC
LIMIT 10
```
NOTE on ILIKE: The agent-id in the ILIKE pattern needs special handling. Use db-escape for the main parameter comparisons, but for the ILIKE, construct `'%@' || {escaped-id} || '%'` where escaped-id is the escaped agent ID string.

Post-process each row: rename :from-agent to :from (to match HTTP endpoint JSON output which uses "from" not "from_agent"). Coerce :id to integer via parse-integer.

**Function: `db-perceive-tasks`** `(pool agent-id)` -> vector of hash-tables
First query agent metadata to determine routing:
```sql
SELECT department, agent_tier,
       tool_scope IS NOT NULL AND array_length(tool_scope, 1) > 0 as has_tools
FROM agents WHERE id = {escaped-id}
```
Then use the appropriate task query based on agent type:
- If agent_id is "lara" or "sarah" (triage): unassigned tasks, excluding blocked
- If is_exec (agent_tier = 'executive'): personal + department unassigned, excluding blocked
- If has_tools but not exec/triage: personal tasks only, excluding blocked
- If no tools and not triage: empty vector

All task queries include these columns: `id, LEFT(text, 300) as text, status, assignee, array_to_json(assigned_to) as assigned_to, department, stage, goal_id, stage_notes::text, project_id, source, context, parent_id, priority, scheduled_at::text, array_to_json(blocked_by) as blocked_by`

The blocked_by exclusion clause for all variants:
```sql
AND (blocked_by IS NULL OR blocked_by = '{}' OR NOT EXISTS (
  SELECT 1 FROM unnest(blocked_by) AS dep_id
  JOIN tasks t ON t.id = dep_id
  WHERE t.status NOT IN ('done', 'completed')
))
```

Post-process: coerce :id, :goal-id, :project-id, :parent-id to integer. Parse :stage-notes from JSON string to hash-table. Parse :assigned-to and :blocked-by from JSON arrays.

**Function: `db-perceive-blocked-tasks`** `(pool agent-id)` -> vector of hash-tables (exec only)
```sql
SELECT t.id, LEFT(t.text, 200) as text, t.status,
       array_to_json(t.blocked_by) as blocked_by, t.assignee, t.project_id
FROM tasks t
WHERE t.project_id IN (SELECT id FROM projects WHERE owner = {escaped-id})
  AND t.blocked_by IS NOT NULL AND t.blocked_by != '{}'
  AND EXISTS (
    SELECT 1 FROM unnest(t.blocked_by) AS dep_id
    JOIN tasks bt ON bt.id = dep_id
    WHERE bt.status NOT IN ('done', 'completed')
  )
ORDER BY t.id DESC LIMIT 10
```

**Function: `db-perceive-critical-issues`** `(pool agent-id)` -> vector of hash-tables (exec only)
```sql
SELECT t.id, LEFT(t.text, 200) as text, t.stage_notes::text, t.project_id, t.assignee
FROM tasks t
WHERE t.project_id IN (SELECT id FROM projects WHERE owner = {escaped-id})
  AND t.status IN ('done', 'completed')
  AND t.stage_notes IS NOT NULL
  AND t.stage_notes -> 'issues' IS NOT NULL
  AND EXISTS (
    SELECT 1 FROM jsonb_array_elements(t.stage_notes -> 'issues') AS issue
    WHERE issue->>'severity' = 'CRITICAL'
  )
ORDER BY t.updated_at DESC LIMIT 10
```

**Function: `db-perceive-documents`** `(pool agent-id since)` -> vector of hash-tables
```sql
SELECT d.id, d.title
FROM documents d
JOIN agent_document_links adl ON adl.document_id = d.id
WHERE adl.agent_id = {escaped-id}
  AND COALESCE(d.modified_at, d.created_at) > {escaped-since}::timestamptz
ORDER BY COALESCE(d.modified_at, d.created_at) DESC LIMIT 5
```

**Function: `db-perceive-team-activity`** `(pool agent-id tier since)` -> vector of hash-tables
Only for tier = "working" or "prime". Query direct reports:
```sql
SELECT id FROM agents WHERE reports_to = {escaped-id} AND status = 'active'
```
Then if reports exist:
```sql
SELECT agent_id as agent, action_taken as action, energy_after as energy, tick_at
FROM tick_log WHERE agent_id = ANY(ARRAY[{report-ids}]) AND tick_at > {escaped-since}::timestamptz
ORDER BY tick_at DESC LIMIT 10
```
For the ANY(ARRAY[...]) construct, join escaped report IDs with commas.
NOTE: The result keys must be :agent, :action, :energy (not :agent-id, :action-taken, :energy-after) to match the HTTP endpoint JSON output. Use column aliases in SQL.

**Function: `db-perceive-proactive`** `(pool agent-id has-tools messages-empty tasks-empty)` -> boolean

```sql
SELECT agent_tier FROM agents WHERE id = {escaped-id}
```
```sql
SELECT MAX(created_at)::text as last_acted FROM conversations WHERE from_agent = {escaped-id}
```
Logic: if no tools, return nil. If energy >= 40, cooldown passed (exec: 5 min, staff: 30 min based on agent_tier), and messages-empty and tasks-empty, return t.

**Function: `db-perceive-responsibilities`** `(pool agent-id)` -> hash-table

**Function: `db-perceive`** `(pool agent-id tier since)` -> hash-table
This is the main orchestrator. Calls all sub-functions and assembles the perception hash-table:
1. `db-perceive-messages` -> :messages
2. `db-perceive-tasks` -> :tasks (also returns is-exec and has-tools as secondary values)
3. `db-perceive-documents` -> :documents
4. `db-perceive-team-activity` (only if tier is "working" or "prime") -> :team-activity
5. `db-perceive-proactive` -> :proactive-eligible
6. If proactive-eligible, `db-perceive-responsibilities` -> :responsibilities (else empty HT)
7. `db-perceive-relationships` -> :relationships
8. :requests -> empty vector (disabled, per af64_perception.rs line 459)
9. `db-perceive-recent-memories` -> :recent-memories
10. If is-exec: `db-perceive-blocked-tasks` -> :blocked-tasks and `db-perceive-critical-issues` -> :critical-issues (else empty vectors)

Return the assembled hash-table. Wrap each sub-call in handler-case — if any individual query fails, use empty default (empty vector or empty hash-table) and log the error, but don't fail the entire perception.

**Function: `db-fetch-agents`** `(pool)` -> list of hash-tables (matching /api/agents output)
```sql
SELECT a.id, a.full_name, a.role, a.department,
       array_to_json(a.reports_to) as reports_to,
       a.agent_tier, a.status,
       a.tool_scope IS NOT NULL AND array_length(a.tool_scope, 1) > 0 as has_tools,
       s.energy, s.tier, s.last_tick_at::text,
       s.ticks_at_current_tier, s.ticks_alive, s.metadata::text
FROM agents a
LEFT JOIN agent_state s ON s.agent_id = a.id
ORDER BY a.id
```
Post-process: coerce :energy to float, :ticks-alive and :ticks-at-current-tier to integer, parse :metadata from JSON, parse :reports-to from JSON array. Convert :has-tools from "t"/"f" string to t/nil.

**Function: `db-fetch-fitness`** `(pool agent-id)` -> integer

```sql
SELECT COALESCE(SUM(score), 0) as total
```

Return integer total. On error return 0.
